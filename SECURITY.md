# Безопасность

Если вы нашли уязвимость в `PavelCode`, не публикуйте детали сразу в открытом issue.

## Как сообщить

Отправьте приватное описание проблемы владельцу репозитория `PavelHopson` и приложите:

- краткое описание уязвимости
- шаги воспроизведения
- ожидаемое и фактическое поведение
- версию Node.js, Bun и ОС
- при необходимости минимальный PoC

## Что относится к проекту

В область действия входят:

- исходный код из этого репозитория
- CLI-пакет `@pavelhopson/pavelcode`
- скрипты сборки, запуск и документация проекта

## Что не относится

- уязвимости сторонних провайдеров API
- проблемы сторонних npm-пакетов без воспроизводимого влияния на проект
- ошибки конфигурации на стороне пользователя без уязвимости в коде

## Рекомендация по обработке

До публикации исправления желательно:

- подтвердить проблему локально
- подготовить минимальный патч
- проверить, не раскрываются ли секреты, токены и приватные пути

После исправления можно оформить публичный advisory или issue с описанием уже закрытой проблемы.
